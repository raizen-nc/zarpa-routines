# zarpa-seo-researcher — Гід для Research Routine

Цей Skill завантажується Research Routine перед дослідженням теми для сторінки "Ми лікуємо".

Перед читанням цього Skill вже завантажений: `zarpa-clickup-ops` (операційний гід).

---

## ⚠️ GOTCHAS

1. **Ніколи "фізіотерапевт"** — завжди "фізичний терапевт". Це юридична різниця в Україні.
   Конкуренти часто використовують неправильну термінологію — не копіювати.

2. **Аналізуй українські сайти**, не англомовні. Пошукова видача zarpa.io — Україна/Львів.
   Конкуренти: українські реабілітаційні центри та медичні сайти (МОЗ, ДУ "Інститут...", приватні клініки Львова, Києва).

3. **Не вигадуй статистику.** Якщо немає точних даних — пиши "згідно з даними МОЗ України [рік]" або позначай `[потрібна перевірка]`.

4. **Збережи дослідження через append до description** — не перезаписуй, не пиши в comment.
   Формат: `## 🔍 SEO Research` блок.

5. **Після збереження:** встанови write_now = true І статус = "writing text".
   Саме зміна write_now = true запускає Writing Routine через ClickUp Automation.

## 1. Підготовка до Дослідження

Отримай з description таску (після парсингу Quill Delta):
- [ ] Тему сторінки (назву стану/хвороби)
- [ ] URL slug (наприклад `/we-treat/skolioz`)
- [ ] Цільову аудиторію (батьки дітей, дорослі, родичі пацієнтів)
- [ ] Будь-які вже відомі ключові слова або структуру

Якщо description мінімальний — дослідження будується тільки з назви таску.

---

## 2. Стратегія Дослідження — 15-25 Запитів

### Блок 1: Пошуковий інтент і семантика (5-8 запитів)

```
[стан] симптоми
[стан] лікування у Львові
[стан] фізична терапія
[стан] у дітей (якщо дитяча тема)
як лікувати [стан] вдома
[стан] причини
[стан] реабілітація
```

Також шукай українські варіанти написання:
- "сколіоз" і "сколіозу", "при сколіозі"
- Розмовні форми: "горбиться дитина", "кривошия у немовляти"
- Питання батьків: "що робити якщо у дитини [стан]"

### Блок 2: Аналіз конкурентів (5-8 запитів)

```
site:moz.gov.ua [стан]
[стан] центр реабілітації Львів
[стан] фізична терапія Україна
[стан] лікування site:.ua
реабілітаційний центр [стан] Київ
```

Для кожного конкурента що з'явився — проаналізуй:
- H1 і мета-тег
- Структура сторінки (які H2 використовуються)
- Обсяг тексту (приблизно)
- Чи є FAQ, чи є блок "протипоказання"
- Контентні прогалини: що вони НЕ покривають

### Блок 3: AEO — "People Also Ask" і пов'язані питання (5-8 запитів)

```
[стан] що це таке
як визначити [стан] у дитини
[стан] скільки сеансів потрібно
[стан] чи можна вилікувати повністю
[стан] ціна лікування Україна
[стан] гімнастика вправи
[стан] чи боляче лікувати
```

Записуй реальні питання з Google (підказки в пошуку, People Also Ask, Answer The Public структура).
Ці питання → основа для FAQ секції на сторінці.

### Блок 4 (опціонально): Apify для глибокого аналізу

Якщо потрібна структура конкретних сторінок конкурентів:
```
Apify Actor: apify/web-scraper або crawlee/web-scraper
URL: [URL конкурента]
Витягти: H1, H2, H3, meta description, word count
```

Використовуй Apify тільки якщо WebSearch не дає достатньо структурної інформації.

## 3. Семантичне Ядро — Що Зібрати

Для кожного ключового слова визначити:

| Ключове слово | Тип запиту | Інтент | Пріоритет |
|--------------|-----------|--------|-----------|
| сколіоз лікування Львів | geo-комерційний | записатись | 🔴 Головне |
| сколіоз у дітей симптоми | інформаційний | дізнатись | 🟡 Важливий |
| як лікувати сколіоз фізична терапія | інформаційний | зрозуміти метод | 🟡 Важливий |
| сколіоз 1 ступінь лікування | інформаційний | конкретна ситуація | 🟢 Додатковий |

**Типи інтенту:**
- geo-комерційний → пацієнт хоче записатись у Львові
- інформаційний → хоче зрозуміти стан / методи лікування
- навігаційний → шукає конкретний центр
- транзакційний → хоче ціну / записатись зараз

Пріоритизуй: головне ключове слово (1-2 з найвищою частотністю і гео-намір) + 5-10 вторинних.

```

H2: Фізична терапія при [стан] у Zarpa — Львів (E-E-A-T блок)
```

Адаптуй структуру якщо конкуренти мають інший підхід або тема вимагає додаткових секцій.

## 6. Завершення — Передача Writing Routine

Після того як блок збережено в description:

**Крок 1:** Встанови custom field `write now` = true
```
clickup_update_task(
  task_id="<id>",
  custom_fields=[{"id": "3eb51e48-1787-4146-b84f-eac2fffb65b2", "value": true}]
)
```

**Крок 2:** Зміни статус на "writing text"
```
clickup_update_task(task_id="<id>", status="writing text")
```

Зміна write_now = true → ClickUp Automation → Webhook → Writing Routine запускається.
